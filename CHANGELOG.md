### Fixed
- **Prvi produkcijski build na Vercelu pukao** (`TypeError: Invalid URL`, `Failed to collect page data for /_not-found`) — `NEXT_PUBLIC_SITE_URL` u Vercel Environment Variables bio je postavljen kao gola domena (`kam-ve-iti.vercel.app`) bez `https://` prefiksa; `new URL(...)` u `layout.tsx` (metadataBase) to odbija. Popravljeno postavljanjem pune vrijednosti (`https://kam-ve-iti.vercel.app`) i redeployem. `.env.example` dobio eksplicitno upozorenje o obaveznom protokolu da se ne ponovi.
- GitHub secret scanning blokirao push jer je stvarni Supabase secret key greškom završio u `.env.example` umjesto `.env.local` — očišćeno, ključ nikad nije stigao na remote.
- `.env.local` sadržavao dupliciranu/spojenu verziju varijabli (stari prazan predložak zalijepljen ispod novih vrijednosti) — kod dupliciranih `.env` varijabli zadnja definicija pobjeđuje, pa je prazna verzija odozdo prepisivala stvarni URL. Ubuduće: `.env.local` se puni SAMO stvarnim vrijednostima, bez ostataka predloška.
- **Bug otkriven pri izradi loading skeletona (Faza 8, Dan 3):** `<main>` element unutar `loading.tsx` fallbacka sudara se sa stvarnim `<main>` u stranici koja se streamа — u ovoj verziji Next.js/Turbopack dev servera (16.2.10) to trajno blokira zamjenu fallbacka stvarnim sadržajem (stranica ostaje zaglavljena na skeletonu, potvrđeno uživo kroz više neovisnih testova: čist restart servera, nova kartica, prazan router cache). Rješenje: fallback koristi `<div>` s istim izgledom — ujedno i ispravnije pristupačnosti, jer se izbjegava privremeno dupliciranje `<main>` landmarka.
